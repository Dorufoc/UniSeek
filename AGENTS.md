# UniSeek 项目统一共识

## 1. 语言规范
- **所有注释**：必须使用中文书写
- **UI 文本内容**：必须使用中文（涉及国际化需求时除外）
- **沟通语言**：回答任何问题、编写任何说明文档，均使用中文

## 2. Git 提交规范
- 禁止自动执行 `git commit` 操作
- 每次 Git 操作前，必须先向用户说明变更内容，获得用户明确回复确认后，方可执行
- 禁止自动 `git push`，推送前同样需用户确认

## 3. 环境配置
| 环境        | 版本    | 说明                               |
| ----------- | ------- | ---------------------------------- |
| Java        | 1.8     | 后端开发，对应 `uniseek_java` 模块 |
| Vue         | 3       | 前端开发，对应 `uniseek_vue` 模块  |
| ArkTS       | 6.1.1   | 鸿蒙开发，对应 `uniseek_arkts` 模块|
| MySQL       | 8       | 数据库                             |

## 4. 工作路径规范
- **禁止逃逸工作路径**：所有代码编写必须在对应功能模块目录下进行
- 各模块职责分明，不得跨模块编写代码
- ArkTS 代码 → `uniseek_arkts/` 目录下
- Java 代码 → `uniseek_java/` 目录下
- Vue 代码 → `uniseek_vue/` 目录下

## 5. 前后端分离
- 本项目采用前后端分离架构
- **前端（Vue 3）**：只关注 UI 展示、用户交互、API 调用
- **后端（Java 1.8）**：只关注业务逻辑、数据处理、API 提供
- **鸿蒙端（ArkTS 6.1.1）**：独立的前端应用
- 严禁在前后端代码中混杂对方领域的逻辑

## 6. 目录设计与代码规范
- 项目目录设计遵循企业级标准
- 采用**分层架构设计**（Controller / Service / DAO / Entity 等）
- 代码语法遵循各语言官方最佳实践
- 各层职责清晰，禁止跨层调用混乱
- 命名规范统一，结构清晰可维护

## 7. 提问机制
- 遇到以下情况，必须及时调用 `AskUserQuestion` 工具向用户提问：
  - 需求不明确或存在歧义
  - 需要用户做决策选型
  - 操作可能带来影响（如删除、覆盖、重构等）
  - 不确定是否符合用户预期

## 8. 在线搜索
- 遇到不确定的技术问题、最佳实践、API 用法等，及时使用 `WebSearch` 工具进行在线搜索验证，确保方案准确性

## 9. 任务管理
- 复杂任务（3 步以上）必须使用 `TodoWrite` 工具创建任务列表，跟踪进度
- 完成任务及时标记，确保每个步骤不遗漏
